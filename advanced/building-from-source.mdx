## Prerequisites

Building Flashlight requires three main tools:

1. **Rust** - Backend language
2. **Node.js** - Frontend tooling
3. **pnpm** - Package manager (required, not npm/yarn)

### Install Rust

Install Rust using rustup (official installer):

```bash
# macOS, Linux, or WSL
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Windows (download and run)
# https://rustup.rs/
```

**Minimum Rust version:** 1.77.2 (specified in `Cargo.toml`)

Verify installation:

```bash
rustc --version
# rustc 1.77.2 (or newer)

cargo --version
# cargo 1.77.0 (or newer)
```

### Install Node.js

Flashlight requires Node.js for frontend build tooling:

```bash
# Using nvm (recommended)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 20
nvm use 20

# Or download from nodejs.org
# https://nodejs.org/
```

**Minimum Node.js version:** 18.0.0 or newer (20+ recommended)

Verify installation:

```bash
node --version
# v20.x.x or newer
```

### Install pnpm

<Warning>
Flashlight uses **pnpm**, not npm or yarn. Always use pnpm for all package operations.
</Warning>

Install pnpm globally:

```bash
# Using npm
npm install -g pnpm

# Using Homebrew (macOS)
brew install pnpm

# Using standalone script
curl -fsSL https://get.pnpm.io/install.sh | sh -
```

Verify installation:

```bash
pnpm --version
# 10.19.0 (or newer)
```

**Package manager version:** Locked to `pnpm@10.19.0` in `package.json`

## Platform-Specific Requirements

### macOS

**Required:**
- Xcode Command Line Tools

```bash
xcode-select --install
```

**Supported architectures:**
- Apple Silicon (arm64)
- Intel (x86_64)
- Universal binary (both)

### Windows

**Required:**
- Microsoft Visual C++ Build Tools
- Windows 10 SDK

Install via Visual Studio Installer:
1. Download [Visual Studio 2022](https://visualstudio.microsoft.com/downloads/)
2. Select "Desktop development with C++" workload
3. Ensure "Windows 10 SDK" is checked

**Optional (for MFT support):**
- Administrator privileges (for NTFS MFT enumeration and USN Journal)

## Installing Dependencies

Install Node.js dependencies with pnpm:

```bash
pnpm install
```

This installs:
- SvelteKit and Svelte 5
- Tauri CLI
- Tailwind CSS and UI components
- Development tools (TypeScript, Vite, etc.)

<Note>
Rust dependencies are automatically downloaded by Cargo during the first build.
</Note>

## Development Mode

Run Flashlight in development mode with hot-reload:

```bash
pnpm tauri dev
```

This command:
1. Compiles the Rust backend (first run takes 5-10 minutes)
2. Starts the Vite dev server for frontend
3. Opens the Tauri window
4. Enables hot-reload for both frontend and backend

**Alternative: Frontend only (for UI development):**

```bash
pnpm dev
```

Runs the SvelteKit dev server without Tauri (no backend functionality).

### Development Build Times

| Component | First Build | Incremental |
|-----------|-------------|-------------|
| Rust backend | 5-10 minutes | 10-30 seconds |
| Frontend | 2-5 seconds | &lt;1 second |
| Full rebuild | 5-10 minutes | 10-30 seconds |

<Info>
Rust compilation is slow the first time due to dependency compilation. Subsequent builds are much faster thanks to incremental compilation.
</Info>

## Production Build

Build optimized production binaries:

```bash
pnpm tauri build
```

This command:
1. Builds the frontend with Vite (production mode)
2. Compiles the Rust backend with `--release` flag
3. Creates platform-specific installers
4. Generates code signatures (if configured)

### Build Output Locations

**macOS:**
```
src-tauri/target/release/bundle/
├── dmg/
│   └── Flashlight_x.x.x_universal.dmg     # Universal binary (Intel + Apple Silicon)
└── macos/
    └── Flashlight.app
```

**Windows:**
```
src-tauri\target\release\bundle\
├── msi/
│   └── Flashlight_x.x.x_x64_en-US.msi     # MSI installer
└── nsis/
    └── Flashlight_x.x.x_x64-setup.exe     # NSIS installer
```

**Linux:**
```
src-tauri/target/release/bundle/
├── deb/
│   └── flashlight_x.x.x_amd64.deb         # Debian package
└── appimage/
    └── flashlight_x.x.x_amd64.AppImage    # Portable AppImage
```

### Build Times

| Platform | Debug Build | Release Build |
|----------|-------------|---------------|
| macOS (M1/M2) | 2-3 minutes | 5-8 minutes |
| Windows | 3-5 minutes | 8-12 minutes |
| Linux | 3-5 minutes | 8-12 minutes |

<Note>
Release builds are optimized for size and speed. Expect longer compile times but significantly better runtime performance.
</Note>

## Building the CLI Tool

Flashlight includes a CLI tool (`fl`) that can be built separately:

```bash
cd src-tauri
cargo build --release --bin fl
```

**Output:** `src-tauri/target/release/fl` (or `fl.exe` on Windows)

### Install CLI Globally

```bash
# Copy to system PATH
sudo cp target/release/fl /usr/local/bin/

# Or run the built-in setup
./target/release/fl init
```

The `fl init` command:
- Adds `fl` to your PATH
- Installs shell completions (zsh/bash/fish)
- Creates necessary directories

## Troubleshooting

### "rustc not found" or "cargo not found"

**Solution:** Ensure Rust is installed and in PATH:

```bash
source $HOME/.cargo/env
# Or restart your terminal
```

### "pnpm: command not found"

**Solution:** Install pnpm globally:

```bash
npm install -g pnpm
```

### Tauri build fails on Linux

**Solution:** Install WebKit and other system dependencies:

```bash
sudo apt install libwebkit2gtk-4.1-dev libssl-dev
```

### "Linking with cc failed" on macOS

**Solution:** Install Xcode Command Line Tools:

```bash
xcode-select --install
```

### Windows build fails with "LINK: fatal error"

**Solution:** Install Visual Studio Build Tools with C++ workload:

1. Download [Visual Studio Installer](https://visualstudio.microsoft.com/downloads/)
2. Select "Desktop development with C++"
3. Restart terminal after installation

### Build is very slow

**Solutions:**

1. **Enable parallel compilation** (already default in Cargo)
2. **Use faster linker** (mold on Linux, lld on Windows):

```toml
# .cargo/config.toml
[target.x86_64-unknown-linux-gnu]
linker = "clang"
rustflags = ["-C", "link-arg=-fuse-ld=mold"]
```

3. **Reduce optimization** for debug builds:

```toml
# Cargo.toml
[profile.dev]
opt-level = 1  # Basic optimizations
```

## Continuous Integration

Flashlight uses GitHub Actions for CI/CD:

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
      - uses: actions/setup-node@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: pnpm install
      - run: pnpm tauri build
      # Upload artifacts...
```

**Automated builds for:**
- macOS (universal binary)
- Windows (x64)
- Linux (deb + AppImage)
